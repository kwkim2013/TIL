```python id="c4t8bn"
import random

user_wins = 0
com_wins = 0

dic_rps_ref = {
    '가위': {'가위': None, '바위': '컴','보': '사'},
    '바위': {'가위': '사', '바위': None,'보': '컴'},
    '보': {'가위': '컴', '바위':'사','보': None},
}
# Define computer's choices as strings for comparison with user_rps
list_com_choices = ['가위', '바위', '보']

print("가위바위보 게임을 시작합니다! 먼저 3승을 하는 사람이 이깁니다.")

while user_wins < 3 and com_wins < 3: # Continue until one player wins 3 rounds
    user_rps = None
    while True: # Loop for valid user input
        user_rps = input("가위, 바위, 보 중에서 하나를 선택/입력해 주세요 : ")
        if user_rps in ['가위','바위','보']:
            print(f"사용자가 선택한 것은 {user_rps} 입니다.")
            break
        print("사용자가 잘못 선택했어요. 다시 입력해 주세요.")

    # Computer makes a choice
    com_rps_index = random.randint(0, 2)
    com_rps = list_com_choices[com_rps_index]
    print(f"컴퓨터가 선택한 것은 {com_rps} 입니다.")

    # Determine the winner of the current round
    winner = dic_rps_ref[user_rps][com_rps]

    if winner == '사':
        user_wins += 1
        print("이번 라운드는 사용자가 이겼습니다!")
    elif winner == '컴':
        com_wins += 1
        print("이번 라운드는 컴퓨터가 이겼습니다!")
    else: # winner is None (draw)
        print("이번 라운드는 비겼습니다!")

    print(f"현재 점수: 사용자 {user_wins} : 컴퓨터 {com_wins}")
    print("-" * 30) # Separator for rounds

# After the game ends (one player reached 3 wins)
if user_wins == 3:
    print("축하합니다! 사용자가 최종 승리했습니다! 🎉")
else:
    print("아쉽지만 컴퓨터가 최종 승리했습니다. 🤖")

```

### 💻 학습 성과

- `while` 반복문을 활용하여 사용자 입력을 검증하고, 올바른 값이 입력될 때까지 반복 처리하는 방법을 익혔다.
- 조건문과 반복문을 함께 사용하여 프로그램의 전체 흐름을 제어하는 능력을 향상시켰다.
- 딕셔너리 자료구조를 활용해 가위바위보 승패 로직을 효율적으로 구현할 수 있게 되었다.
- `random` 모듈을 사용하여 컴퓨터의 선택을 무작위로 생성하는 방법을 이해하였다.
- 점수 누적 및 종료 조건을 설정하여 하나의 완성된 게임 프로그램을 구현할 수 있었다.
- 단순한 코드 작성이 아닌, 문제 해결을 위한 로직 설계의 중요성을 이해하게 되었다.

### 💡 느낀 점 / 이해

이번 수업을 통해 가위바위보 게임을 직접 구현하면서 단순한 조건문을 넘어서 전체적인 **프로그램 흐름 제어**의 중요성을 이해할 수 있었다. 특히 `while` 반복문을 활용하여 사용자 입력을 계속 검증하고, 올바른 값이 들어올 때까지 반복하는 구조가 인상적이었다.

또한 딕셔너리를 이용해 승패 로직을 구성한 부분이 매우 효율적이라고 느꼈다. 기존에는 `if-else` 문을 여러 번 사용해야 할 것이라고 생각했지만, 자료구조를 활용하면 훨씬 간결하고 가독성 높은 코드를 작성할 수 있다는 점을 배웠다.

랜덤 함수를 통해 컴퓨터의 선택을 구현하면서 실제 게임처럼 동작하는 프로그램을 만들 수 있었고, 점수 누적과 종료 조건 설정을 통해 하나의 완성된 프로그램을 만드는 과정이 흥미로웠다.

전체적으로 이번 학습을 통해 단순 문법이 아닌 **로직 설계 능력과 문제 해결 능력**이 중요하다는 것을 느꼈고, 앞으로는 코드를 작성하기 전에 흐름을 먼저 설계하는 습관을 가져야겠다고 생각했다.
